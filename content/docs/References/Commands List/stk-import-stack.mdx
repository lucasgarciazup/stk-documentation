O comando **`stk import stack`** é responsável por adicionar uma stack ao seu projeto. 

### **Como funciona?**
Quando você roda o comando, tanto os templates quanto os plugins da stack criada são disponibilizados para serem usados nos comandos **`stk create app`**, **`stk create env`** e **`stk apply plugin`**.

<alert color="secondary">
Se você importou uma stack de um repositório privado ou interno, você precisa re-autenticar o Git para todos os plugins da sua stack.

Você pode configurar o Git para armazenar sua senha com o comando abaixo e deixar isso automático.

 `git config --global credential.helper store`

Para mais informações, acesse a [**documentação do Git**](https://git-scm.com/book/pt-br/v2/Git-Tools-Credential-Storage).
</alert>


### **Parâmetros**
Confira a opção disponível no comando:

| Parâmetro | Descrição |
| ----------- | ----------- |
| `STACK_URL` | Informa a URL do repositório Git de uma stack. |


### **Exemplo**
Veja um exemplo abaixo:

1. Adiciona os plugins e templates publicados na stack para o uso da CLI:
```
stk import stack [URL do git]
```
