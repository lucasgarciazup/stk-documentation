# **O que há de novo?**
A nova versão traz novas atualizações como melhorias de performance no comando de atualizar as stacks e remoção das stacks iniciais do [<ins>**stk init**</ins>]({{< ref path="/docs/References/Commands List/stk-init.mdx" >}}) e divulgação do [<ins>**portal de studios**</ins>](https://www.stackspot.com.br/studios).

## **Features**
- **Monorepo:** Você pode criar templates e plugins no mesmo repositório da stack sem necessidade de vários repositórios. E a possibilidade de stacks mistas com e sem plugins em repositórios separados.
- **Novos comandos:**  [<ins>**stk delete plugin**</ins>]({{< ref path="/docs/References/Commands List/stk-delete-plugin.mdx" >}}) e [<ins>**stk delete template**</ins>]({{< ref path="/docs/References/Commands List/stk-delete-template.mdx" >}}).
- **Stackfiles:** Você pode criar, listar stackfiles de uma ou todas stacks e criar um app/env baseado em um Stackfile. Para mais informações, veja a seção [<ins>**stk create stackfile**</ins>]({{< ref path="/docs/References/Commands List/stk-create-stackfile.mdx" >}}).


## **Mudanças**
- Melhorias de performance no comando de [<ins>**stk update stack**</ins>]({{< ref path="/docs/References/Commands List/stk-update-stack.mdx" >}}).


## **Resolução de problemas (Bug Fixes)**
- Autenticidade (notarize) Apple de Apps dos pacotes PKG. O instalador de MacOS está adequado ao processo de notarize.
- Resolução do problema no parse de parâmetros ao executar [**`stk import stack`**]({{< ref path="/docs/References/Commands List/stk-apply-plugin.mdx" >}}) usando **`--plugin-path`** e passando parâmetros na linha de comando.=======
- Ajustes no [**`--help`**] dos comandos de delete.
- Padronização da mensagem de Git init da stack.
- Remoção das configs dos shells no [<ins>**Uninstall**</ins>]({{< ref path="/docs/STK CLI/Uninstall/uninstall.mdx" >}}). 
- Validação do formato da URL para importar stacks.
- Aplicação de validações que são feitas quando os inputs são aplicados nos valores resolvidos por meio dos parâmetros.
