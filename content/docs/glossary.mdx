Veja abaixo o glossário da StackSpot:

Reponsáveis pela criação e uso de Templates/Plugins/app/env dentro do STK CLI.
## **Criadores (Creators)**
É quem desenha e cria Stacks para produção de softwares. E logo depois, publica Templates e Plugins nessas Stacks que podem ser usadas pelos estúdios. 

É o criador quem define os aspectos da Stack como: linguagem de programação, bibliotecas e componentes, fases do ciclo de vida, controle de qualidade, testes, empacotamento, distribuição e boas práticas implementadas pela Stack.

Ao publicar um studio de Stack, o criador **deve garantir a existência de Templates e Plugins que são compatíveis entre si**. Além disso, ele também deve garantir que quando os Templates e Plugins forem aplicados em conjunto, eles sejam capazes de produzir projetos de software de alta qualidade, de forma rápida e eficiente, abstraindo dos usuários as complexidades do processo de desenvolvimento.

Cada Stack pode ter objetivos diferentes e endereçar diferentes tipos de produtos. É o criador quem define a estratégia de desenho de suas Stacks para que os seus usuários consigam atingir seus objetivos, com uma baixa carga cognitiva e alta qualidade.

O criador cria Templates por meio do comando [<ins>**`stk create template`**</ins>]({{< ref path="/stk-create-template.md" >}}), e eles são a base de um projeto para o usuário utilizar.

Os Templates podem ser de linguagens específicas e, com isso, podem utilizar os Plugins compatíveis com essa mesma linguagem.

## **Usuário** 
É a pessoa (usuário), ou organização, que consome as Stacks produzidas e publicadas pelo criador. O usuário deve seguir os padrões propostos pelo criador para diminuir o tempo de desenvolvimento e aumentar a qualidade dos seus produtos.

Ao usar o STK CLI, o usuário consegue importar as Stacks publicadas pelos criadores e iniciar, com facilidade, novos projetos baseados nos Templates das Stacks produzidas.

Estes projetos criados com os Templates podem ser ampliados com Plugins, que adicionam novas funcionalidades e recursos ao projeto.

## **Ambiente (env)**
É um conjunto de recursos e configurações compartilhados entre uma ou mais aplicações.

>Aplicações podem usar ou não os recursos compartilhados por um ambiente.

Um ambiente pode ser exemplificado pela criação de um runtime para containers (como um cluster [<ins>**ECS**</ins>](https://docs.aws.amazon.com/pt_br/AmazonECS/latest/developerguide/clusters.html) ou [<ins>**EKS**</ins>](https://docs.aws.amazon.com/pt_br/eks/latest/userguide/clusters.html)), de um [<ins>**RDS**</ins>](https://aws.amazon.com/pt/rds/) que permite ser compartilhado entre várias aplicações ou pela criação de recursos de rede compartilhados entre as várias aplicações de uma empresa, como [<ins>**VPCs**</ins>](https://aws.amazon.com/pt/vpc/), subnets etc.

## **Aplicação (app)**
É um produto ou módulo de um produto que implementa alguma funcionalidade. Uma aplicação pode ser, por exemplo, uma biblioteca, API, um aplicativo mobile, microsserviço ou outro componente de software que disponibiliza funcionalidades importantes de negócio.

A partir do tipo da aplicação, ela pode fazer referência a um ambiente para usar os recursos compartilhados dele. Mas uma aplicação também pode ser criada sem fazer referência a um ambiente. Resumindo: essa referência é opcional.

## **Comandos STK CLI**
No STK CLI, os comandos são divididos em três categorias básicas: Core, Criadores e Usuários.
- Os **comandos Core** são usados tanto pelos criadores quanto pelos usuários de Stacks;
- Os **comandos para Criadores** são utilizados na criação de Stacks, Templates e Plugins;
- Os **comandos para Usuários** são voltados para o consumo das Stacks produzidas. É possível adicionar Stacks, usar Templates e Plugins para criar aplicações e ambientes.
Para mais informações, acesse a seção [<ins>**Comandos**</ins>]({{< ref path="STK CLI/Commands.md">}}).

## **Stack**
É o conjunto de Templates, Plugins e [<ins>**Stackfiles**</ins>]({{< ref path="Creators Guide/howto-create-stackfiles.md">}}) usados para criar [**aplicações (app)**]({{< ref path="/glossary.md#aplicação-app" >}}) e [**ambientes (env)**]({{< ref path="/glossary.md#ambiente-env" >}}). Uma Stack é um repositório Git criado pelo comando [<ins>**`stk create stack`**</ins>]({{< ref path="/stk-create-stack.md" >}}) do STK CLI. Nele, os Plugins e Templates criados pelo estúdio podem ser adicionados por meio dos comandos [<ins>**stk add plugin**</ins>]({{< ref path="/stk-add-plugin.md" >}}) ou [<ins>**stk add template**</ins>]({{< ref path="/stk-add-template.md" >}}) dentro da pasta da Stack. 

Dentro dela, os Plugins e Templates viram Plugins em repositórios separados. Eles podem ser versionados de forma individual e mantidos por times diferentes que não necessariamente tenham acesso ao repositório da Stack.

## **Template**
É um modelo de aplicação ou ambiente que você aplica para construir um projeto. O Template recebe um conjunto de variáveis que, quando são preenchidas, produzem a estrutura de um projeto.
Essas variáveis são: linguagem de programação, build system, frameworks, empacotamento e outros componentes estruturais usados para construir uma Stack.

Em resumo: o Template deve definir todas as capacidades básicas e padronizadas de uma Stack, que depois poderão ser estendidas por meio de Plugins.

De acordo com a estratégia que o estúdio quer adotar na Stack, um Template pode ser complexo ou básico. Dependendo dos tipos definidos no Template, ele pode ser usado nos comandos[<ins>**`stk create app`**</ins>]({{< ref path="/stk-create-app.md" >}}) e/ou [<ins>**`stk create env`**</ins>]({{< ref path="/stk-create-env.md" >}}) para criar uma Stack.


Cada Template é versionado em um repositório Git e o estúdio publica os Templates em uma Stack que agrupa os Plugins e Templates. O versionamento dos Templates deve aderir ao [**versionamento semântico**](https://semver.org/), pois o STK CLI sempre mantém os Templates na última tag publicada, que pode ser na branch main, ou quando não houver tags, na master.

## **Plugin**
São extensões de uma Stack que permitem a agregação de novas capacidades ao Template base por meio da inclusão de código fonte, importação de bibliotecas e até mesmo a execução de comandos e operações complexas para adicionar capacidades a Stack.

A estrutura de um Plugin é igual a de um Template, com exceção dos tipos, que são **`app`** e **`env`**, no lugar de **`app-template`** e **`env-template`**.

## **Stackfiles**

São bundles, ou seja, um conjunto de Templates e Plugins predefinidos de uma Stack, que serão aplicados nos comandos [**`stk create app`**]({{< ref path="/docs/References/Commands List/stk-create-app.md" >}}) ou [**`stk create env`**]({{< ref path="/docs/References/Commands List/stk-create-env.md" >}}). Ao utilizar um Stackfile, o usuário pode criar um **app** ou **env** e aplicar um bundle de Plugins em um único comando do STK CLI.

Os Stackfiles são úteis para que os criadores de Stack definam dentro da própria Stack combinações comuns de Templates e Plugins, e definam "receitas prontas" para as combinações mais comuns dentro da Stack.

Os usuários podem definir seus próprios [**Stackfiles**]({{< ref path="/docs/Creators Guide/howto-create-stackfiles.md" >}}) para agilizar o processo de criação de projetos que usem um bundle predefinido de Templates e Plugins.
