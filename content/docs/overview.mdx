A **StackSpot** é o lugar para que desenvolvedores acelerem sua produção de software, usando uma [<ins>**Stack**</ins>]({{< ref path="/glossary.md#stack" >}}) pronta para o projeto. Ou para o desenvolvedor ter um **Spot**, um lugar onde ele pode publicar sua própria **Stack** para que outros desenvolvedores a utilizem.  

O **estúdio** cria uma **Stack**, que possui [<ins>**templates**</ins>]({{< ref path="/glossary.md#template" >}}) e [<ins>**plugins**</ins>]({{< ref path="/glossary.md#plugin" >}}).

O [**usuário**]({{< ref path="/glossary.md#usuário-consumidor" >}}) utiliza o STK CLI para adicionar uma Stack e consumir os Templates e Plugins que serão utilizados na criação de [**ambientes (env)**]({{< ref path="/glossary.md#ambiente-env" >}}) e [**aplicações (app)**]({{< ref path="/glossary.md#aplicação-app" >}}).

O StackSpot é dividido em dois contextos, o de **criador** de conteúdo e o **usuário** que utiliza o que foi produzido por meio do **STK CLI** (Interface de Linha de Comando). 

1. O criador é responsável por criar **Stacks**, **Templates** e **Plugins**.
2. O usuário utiliza as stacks para consumir Templates e Plugins que serão utilizados na criação de **ambientes(env)** e **aplicativos (app)**.


### **Exemplo**  
Um **estúdio** pode criar templates de uma Stack, por meio do comando do STK CLI **`stk create template`**, que pode ser a base de um projeto de **API de .net** para o usuário.

Os **Templates** podem servir para diferentes tipos de projeto, além de APIs. Com isso, podem utilizar os Plugins compatíveis com essa mesma linguagem.
